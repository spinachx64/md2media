# 主题（themes）说明

这个目录放的是排版主题，**一套主题 = 一个 `.css` 文件**。文件名（去掉 `.css`）就是主题名，
CLI 用 `-t 主题名` 指定，预览页下拉里也会自动列出来。

目前内置四套：

- `summer-breeze.css`：默认主题，夏日海风风格，清凉明亮，适合夏季长文 / 工具教程 / 产品更新。
- `mint-terminal.css`：薄荷终端风格，适合 CLI / DevOps / 工具类文章。
- `graphite-orange.css`：石墨橙风格，适合实践复盘 / 性能优化 / 工程总结。
- `mdnice-classic.css`：仿 mdnice 经典风格，冷色调。

## 一、mdnice-classic 配色说明

整套走**深蓝冷色调**，克制、偏技术感。主色统一在深蓝 `#2b6cb0` 上，没有撞色的红。

### 主色板

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 正文文字 | `#3f3f3f` | 深灰，主体阅读色 |
| 主强调色 | `#2b6cb0` | 深蓝。h1/h2/h3、加粗、斜体、h2 左色条、引用左条、链接、行内代码都用它 |
| 标题文字 | `#3f3f3f` | h4–h6 使用深灰黑，h1/h2/h3 使用主强调色 |
| 次要文字 | `#6a6a6a` | 引用块内文字，比正文浅 |
| 行内代码底色 | `#eef3fb` | 冷色淡蓝底，配深蓝字 |
| 引用块底色 | `#f7f8fb` | 极浅冷灰 |
| 表头底色 | `#f2f4f8` | 浅灰蓝 |
| 分割线 / 表格边框 | `#e0e0e0` / `#dfdfdf` | 浅灰 |
| 代码块底色 | `#282c34` | 深色（One Dark 背景） |
| 代码块文字 | `#abb2bf` | 浅灰，配深底 |

> 想整体换色系，最快的办法是全局替换主强调色 `#2b6cb0`（它出现在 h1/h2/h3、加粗、斜体、h2 左条、
> 引用左条、链接、行内代码等位置），再顺带调一下行内代码底色 `#eef3fb`，保持深浅协调即可。

| --- | --- | --- |
| 正文 / 段落 | 16px | `#3f3f3f` 深灰 |
| h1 | 24px | `#2b6cb0` 深蓝 |
| h2 | 20px | `#2b6cb0` 深蓝（左条同色） |
| h3 | 17px | `#2b6cb0` 深蓝 |
| h4 | 16px | `#3f3f3f` 黑 |
| h5 | 15px | `#3f3f3f` 黑 |
| h6 | 14px | `#3f3f3f` 黑 |
| 引用块 | 15px | `#6a6a6a` 次要灰 |
| 行内代码 | 14px | `#2b6cb0` 深蓝 |

> 标题层级：h1 居中大标题，h2 带主色左色条，h1/h2/h3 使用主色，h4～h6 是黑色粗体、仅字号逐级递减（16→14px）。

### 代码块语法高亮

深色代码块里的语法高亮用 highlight.js 的 **One Dark** 配色（文件末尾 `.hljs-*` 那一段），
在深底上本来就协调，换主色系时通常不用动它。

## 三、如何新增一套主题

### 1. 复制一份现有主题改名

```bash
cp mint-terminal.css my-theme.css
```

文件名 `my-theme` 就是新主题名，无需在别处注册——CLI 和预览页会自动扫描到。

### 2. 改样式，注意两条硬规则

这两条是微信公众号兼容的底线，改任何主题都要守住：

1. **所有选择器都以 `#nice` 开头。** 文章正文会被包在 `<section id="nice">` 里，主题 CSS
   以 `#nice` 作用域，juice 内联时才只作用于文章本体、不污染别处。
2. **颜色、间距等尽量用固定值（px / hex），不要用 `rem`、CSS 变量、`var()`。**
   微信只认标签上的内联 `style`，juice 内联后变量会失效。

### 3. 关于代码块换行（重要，别删）

代码块的多行换行**不靠 CSS**，而是 `render.mjs` 把每一行代码包成
`<span style="display:block">`（块级元素天然独占一行，微信粘贴时不会合并）。所以：

- `#nice pre code` **千万不要**加 `display: -webkit-box` —— 它会把每行的 block span 横向排列，反而毁掉换行。
- 代码块的字色、底色、字体随便改，但别动这条结构约定。

### 4. 用新主题出稿

```bash
# CLI
node ../build.mjs ../../某篇文章.md -t my-theme

# 或预览页：node ../serve.mjs，在下拉里选 my-theme
```

## 四、常见可改点速查

| 想改什么 | 改哪 |
| --- | --- |
| 整体主色 | 全局替换 `#2b6cb0` |
| 正文字号 | `#nice` 和 `#nice p` 的 `font-size`（两处一起改） |
| 标题风格 | `#nice h1` ~ `#nice h6`（h1/h2/h3 使用主色，h4~h6 统一黑色，仅字号递减） |
| 引用块外观 | `#nice blockquote`（左条 `border-left`、底色 `background`） |
| 代码块配色 | `#nice pre`（底色）+ 末尾 `.hljs-*`（高亮色） |
| 行内代码 | `#nice p code` 那组（字色 + 底色） |
